## 一、CRITICAL — 必须修复

### C1. 图层旋转偏移公式错误

- **文件**: `src/core/fpdocument.pas` L1078 (CW) / L1103 (CCW)
- **类别**: BUG

**现有代码 (CW):**
```pascal
Layer.Surface.Rotate90Clockwise;
Layer.OffsetX := -PreviousOffsetY;
Layer.OffsetY := PreviousOffsetX;
```

**正确公式:**
```pascal
Layer.Surface.Rotate90Clockwise;
Layer.OffsetX := FHeight - PreviousOffsetY - Layer.Surface.Width;
// 注: 旋转后 Surface.Width = 旧的 LayerHeight
Layer.OffsetY := PreviousOffsetX;
```

**CCW 同理:**
```pascal
// 现有:
Layer.OffsetX := PreviousOffsetY;
Layer.OffsetY := -PreviousOffsetX;
// 正确:
Layer.OffsetX := PreviousOffsetY;
Layer.OffsetY := FWidth - PreviousOffsetX - Layer.Surface.Height;
// 注: 旋转后 Surface.Height = 旧的 LayerWidth
```

**推导:**
- CW 旋转: 旧画布点 (x, y) → 新画布点 (OldH-1-y, x)
- 旧图层左上角 (Ox, Oy) 旋转后, 新图层矩形左上角 = (OldH - Oy - OldLayerH, Ox)
- 旋转后 `Surface.Width` = OldLayerH, 所以 `NewOffsetX = FHeight - Oy - Surface.Width`
- 只有图层高度 = 文档高度时 `-Oy` 才碰巧正确

**影响:** 多图层文档旋转后,非全尺寸/有偏移的图层位置完全错位,用户数据损坏。

## 二、HIGH — 应尽快修复

### H1. `DeselectClick` 无 `HasSelection` 守卫

- **文件**: `src/app/mainform.pas` L10733
- **类别**: BUG

```pascal
procedure TMainForm.DeselectClick(Sender: TObject);
begin
  SealPendingStrokeHistory;
  // 缺少: if not FDocument.HasSelection then Exit;
  FDocument.PushHistory(LocalizedAction('Deselect'));
  FDocument.Deselect;
  SyncSelectionOverlayUI(True);
end;
```

Esc deselect 路径有 `FDocument.HasSelection` 守卫,但 Cmd+D 菜单路径没有。无选区时按 Cmd+D 会 push 空 "Deselect" undo 条目,污染 undo 栈。

**修复:** 在 `SealPendingStrokeHistory` 后加 `if not FDocument.HasSelection then Exit;`。

### H2. 锁定图层阻止 `tkMoveSelection`

- **文件**: `src/app/mainform.pas` L12502
- **类别**: BUG

```pascal
if Assigned(FDocument) and (FDocument.LayerCount > 0) and
   FDocument.ActiveLayer.Locked and
   not (FCurrentTool in [tkZoom, tkPan, tkColorPicker, tkSelectRect,
     tkSelectEllipse, tkSelectLasso, tkMagicWand]) then
```

`tkMoveSelection` 只移动选区蒙版(文档级位图),不修改图层像素,应在锁定图层上允许。`tkMovePixels` 被正确阻止(改像素)。

---

### H3. `MedianFilter` 未处理 Alpha 通道

- **文件**: `src/core/fpsurface.pas` L2732-L2790
- **类别**: BUG

只中值滤波 R/G/B,Alpha 保留原像素值。来自不同邻居像素的 RGB 中值与原像素的 Alpha 组合,可能产生 `R > A`,违反预乘不变量。后续 `InvertColors`(计算 `A - R`)会下溢。

**修复:** 同时中值滤波 Alpha 通道,或在写回后 clamp `R/G/B <= A`。
