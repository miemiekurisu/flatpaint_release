# Feature Matrix

## Governing rule
- This matrix is for a macOS paint.net recreation target.
- `paint.net reference baseline` is the authoritative product target.
- GIMP/Krita may inform backend implementation only; they do not redefine UX scope, command flow, or feature naming.

## Explicit gaps still to close for stronger paint.net parity

### High Priority
- **Colors: deeper native picker integration**: The app now uses the system color picker plus a companion palette, but the native picker is still opened as a separate dialog; tighter integrated native-panel customization remains open
- **Icon assets**: The app now uses symbol glyphs, but it still lacks true image-backed/vector-backed icon controls
- **Effects breadth**: Only a small longer-tail subset of paint.net effects remains

### Medium Priority
- **Image-list polish**: The tab-strip surface is live now, but richer paint.net-style thumbnail-list affordances and broader route-level testing still remain
- **Remaining longer-tail effects**: The headline shortlist is implemented now; only the smaller remaining paint.net filters outside this audited set are still open
- **Colors: richer preview polish**: Stacked squares are live now, but the final tray/notch/pointer polish still remains below the target
- **Layers: richer thumbnail interaction**: The list is thumbnail-backed now, but it still lacks thumbnail-first drag/reorder and larger preview affordances
- **Iconography**: Symbol glyphs are live, but exact asset-backed icon buttons across toolbar, palettes, and panels are still missing
- **Viewport resize handles**: Direct drag-handle parity around the canvas is still incomplete

### Low Priority

- Colors: "More »/Less" toggle for palette expansion

- History: Toggle trick (click same entry for before/after)
- Status bar: Editable zoom text box (click-to-type)

- Status bar: Dedicated quick-size toggle icon (separate from zoom label)

- Layers: Visibility checkbox inside Properties dialog

- Toolbar: Per-tool blend mode selector
