# Feature Matrix

## Governing rule
- This matrix is for a macOS paint.net recreation target.
- `paint.net reference baseline` is the authoritative product target.
- GIMP/Krita may inform backend implementation only; they do not redefine UX scope, command flow, or feature naming.

## Current implemented rows
- Workspace shell: launchable editor shell with in-window child utility panels as the primary tool surfaces, a denser top toolbar-like control strip, a dedicated top-right utility-button cluster, fixed paint.net-style default panel placement on each launch, compact draggable `Tools` / `Colors` / `History` / `Layers` child panels inside the workspace, direct palette visibility toggles, per-palette close buttons, reset-layout recovery, edge-snap drop behavior, explicit palette title bars, drag-state tint feedback, visible top/left rulers, a centered viewport for smaller canvases, a segmented bottom status strip with synchronized zoom controls, canvas preview, direct canvas interaction, menu commands, a real settings dialog route, and a repository-local `.app` bundle
- Document tabs: not yet implemented in the current Lazarus GUI source; there is still one in-memory document surface only, so the tab strip, image-list thumbnails, multi-document switching, and related context actions remain open work
- File workflow: native `.fpd` save/load with bound-path Save and Save As, `Open Recent`, print-panel routing, `Open`-as-document flow for native/raster/compatibility files, explicit import-as-layer flow, and broader raster read support; the new FPC rewrite core now also has its own native layered document format, and the current Lazarus GUI source plus the CLI both route through native-doc open/save/export and layer-management paths, with the Lazarus GUI source now also honoring a real bound-path `Save`, explicit `Save As`, a persisted `Open Recent` list, explicit `Close`, visible `Print` routes in both the menu and toolbar, a dedicated `New` image modal with size/resolution/print calculations, unified "all supported" open/import filters that merge project files with supported image formats, and a real `Exit` command, while `Save All Images` and `Acquire` remain open
- Command surface parity: the audit and breakdown are now explicit in `docs/COMMAND_SURFACE_BASELINE.md` and `docs/COMMAND_SURFACE_BREAKDOWN.md`; current code is still short of full one-to-one parity for every visible route, but the current toolbar now exposes crop, deselect, a real utility-icon strip for the core palette/settings/help routes, a zoom percentage chooser, direct grid/ruler view toggles, and live `Curves` / `Resize Image` routes while major visible gaps still include the image-list surface, `Acquire`, and `Save All Images`
- Workspace visual parity: explicitly tracked in `docs/UI_PARITY_AUDIT.md`; current shell is usable but still too far from paint.net, although the latest pass tightened toolbar density, palette title-bar treatment, and the dark workspace/palette color separation
- File workflow: native `.fpd` save/load with bound-path Save and Save As, `Open`-as-document flow for native/raster/compatibility files, explicit import-as-layer flow, and flat-raster export; broader file-menu parity still remains open
- Undo/redo: mutation-backed deterministic history for document edits, while zoom/pan/grid/ruler state now stays out of history; the shared document model now also keeps human-readable action labels aligned with undo/redo snapshots for the current `History` palette
- Layers: add, delete, duplicate, rename, reorder, visibility, opacity, blend mode, merge down, flatten; the FPC rewrite CLI now also exposes native-document layer duplication, deletion, reordering, renaming, visibility toggles, opacity control, and merge/flatten routes, and the Lazarus GUI source now exposes layer opacity in its current layer controls
- Selection tools: the new FPC rewrite core now has real rectangle/ellipse/lasso selection masks plus contiguous magic-wand selection, move-selection, move-selected-pixels, fill-selection, erase-selection, invert-selection, select-all, deselect, and crop-to-selection behavior; the current Lazarus GUI source also now routes rectangle/ellipse/lasso selection, magic wand, Shift-add and Option-subtract combine modes, move-selection, move-selected-pixels, visible selection outlines, and live drag previews, though richer parity and full runtime packaging are still open
- Paint tools: zoom tool, pan tool, crop tool, fill selection, rectangle fill, flood fill, linear gradient, line draw, erase, recolor, clone stamp, sample/pick color, and direct single-tap brush/pencil/eraser behavior; the current Lazarus GUI source now also routes a gradient drag tool, a preset-based zoom chooser, and a zoom click-tool path that uses the same preset ladder with pointer-anchored focal preservation, while fuller zoom-tool parity still remains open
- Draw tools: rectangle stroke, rounded-rectangle stroke, ellipse stroke, freeform-shape outline stroke, and text stamping are live; richer object-style shape parity remains open
- View controls: zoom, pan, transparency preview, pixel-grid, rulers, fit-to-view on demand, no-history interactive zoom for continuous input, and initial fit-to-view behavior for very large opened images; the current Lazarus GUI source now also routes `Zoom In`, `Zoom Out`, `Zoom to Selection`, `Actual Size`, `Fit to Window`, `Pixel Grid`, `Rulers`, and unit selection directly in `View`, exposes a visible `Zoom` tool in the compact tool strip, exposes a top-toolbar zoom percentage chooser plus a bottom-right status-bar slider backed by preset steps, keeps the zoom click-tool on that same preset ladder, supports pointer-anchored `Ctrl` / `Command` wheel zoom, renders a zoom-thresholded pixel grid plus scale-aware top/left ruler bands, and reports unit-aware image and cursor measurements through a segmented status strip, while true native pinch and image-list navigation remain open
- Colors: direct primary/secondary color editing, swap, reset, live hex readout in the UI, and a dedicated child colors utility panel; the current Lazarus GUI source now also exposes one-click swap and black/white reset in the toolbar
- Adjustments: auto-level, invert, grayscale, curves, hue/saturation, levels, black/white, sepia, brightness/contrast, and posterize are currently implemented; the new FPC rewrite core now has shared raster paths for that baseline adjustment set, the CLI already exposes `autolevel` / `autoleveldoc`, and the current Lazarus GUI source now routes auto-level, invert, grayscale, `Curves`, `Hue / Saturation`, `Levels`, `Brightness / Contrast`, and `Posterize` directly for the active layer, with `Brightness / Contrast...` now using a dedicated dual-parameter modal on one combined command path, `Curves...` now using a dedicated gamma modal over the current one-value RGB curve implementation, `Hue / Saturation...` now using a dedicated dual-parameter modal, `Levels...` now using a dedicated four-parameter modal, `Posterize...` now using a dedicated levels modal, and fuller multi-point / per-channel curve editing still remaining open
- Effects: blur, sharpen, outline, deterministic noise; the FPC rewrite core now implements that baseline effect family directly in shared code, the CLI exposes raster and native-document routes for it, and the current Lazarus GUI source now has a real `Effects` menu surface for those commands, with `Blur...` now using a dedicated radius modal while `Add Noise...` is still an explicit prompt-based follow-up to close
- Resize/canvas ops: resize canvas, rotate right, rotate left, rotate 180, flip horizontal, flip vertical, crop, crop to selection; the new FPC rewrite core now has baseline crop, nearest-neighbor resize, bilinear resize, flip, 180-degree rotation, and 90-degree rotation support in shared code, and the current Lazarus GUI source now exposes resize image/canvas plus rotate/flip commands directly in `Image`, with `Resize Image...` now using a dedicated aspect-lock modal and explicit resampling selection
- Clipboard: selection-aware copy/cut plus copy-merged and paste image from pasteboard as a new layer; the FPC rewrite core now also has selection extraction, merged-copy, and paste-as-new-layer behavior in shared code, with CLI routes for exporting selected pixels and pasting an external raster into a new native-document layer, and the current Lazarus GUI source now exposes in-process `Cut`/`Copy`/`Copy Selection`/`Copy Merged`/`Paste` routes through that shared core with preserved selection offsets plus explicit `Paste into New Layer` and `Paste into New Image` menu routes
- Menus/Shortcuts: file, edit, image, view, layers, adjustments, effects commands with primary shortcuts, plus documented Windows-to-macOS translation rules; the current Lazarus GUI source now also matches the standard macOS `Command+S`, `Command+Shift+S`, and `Command+X/C/V` baselines in menus, exposes direct palette toggles on `Command+1...4`, and surfaces the main clipboard actions in the toolbar
- Iconography: the current UI no longer relies only on text for tool discoverability; the current Lazarus toolbar now uses a curated compact symbol set in a paint.net-style tool-family order, and the broader icon-language gap remains open beyond that baseline
- Tool/Config Options: brush size, wand tolerance, recolor tolerance, clone size, shape width, text size, gradient orientation, default text, export defaults; the current Lazarus GUI source now switches the shared top option control between brush-size and wand-tolerance semantics based on the active tool
- Tool palette: the visible `Tools` utility surface now defaults to a narrower two-column stack that is materially closer to the tall paint.net tool window shape than the earlier broad four-column card
- Hidden Sheet Options: JPEG quality, PNG interlace, flattening control, compatibility guidance in export/open flows
- Compatibility IO: flattened `PSD` import is available through the shared image loader; flattened `.xcf` import is now also available for common 8-bit uncompressed/RLE GIMP files (normal visibility/opacity composited into one image), while `.pdn` and `.kra` import paths are still explicit open work rather than completed functionality

## Explicit gaps still to close for stronger paint.net parity
- Finer workspace layout fidelity and panel behavior parity
- Broader image-list parity beyond the new baseline tab strip (next/previous navigation, richer context actions, more paint.net-like thumbnail treatment)
- Deeper file-menu parity beyond the new baseline (`Acquire`, fuller recent-file management, and tighter print workflow polish)
- Full one-to-one implementation and testing of every visible command surface in `docs/COMMAND_SURFACE_BASELINE.md`
- More complete dock/undock behavior for child panels inside the main window
- Drag-time translucency or equivalent anti-obstruction feedback for child-panel movement
- Tighter default palette placement and denser utility-window composition
- Stronger visual density parity for the top toolbar, palettes, and status strip
- Smoother continuous scrolling and general viewport motion polish
- Broader shortcut coverage and more exact menu-command parity
- More advanced adjustment/effect families and edge-case behavior
- Deeper foreign project fidelity beyond the current flattened `PSD` / first-pass `XCF` paths, including still-missing `.pdn` / `.kra` import routes and richer layer-preserving import
- Richer canvas interaction affordances beyond the newly added baseline rulers/grid overlays

## Feature gap policy
- If a paint.net behavior is intentionally deferred, mark it as `Deferred` or describe the exact gap explicitly.
- If a feature is partially implemented, record the missing edge in `docs/PROGRESS_LOG.md`.
- Do not let backend-reference convenience remove or rename a paint.net-facing behavior without documenting the reason.
